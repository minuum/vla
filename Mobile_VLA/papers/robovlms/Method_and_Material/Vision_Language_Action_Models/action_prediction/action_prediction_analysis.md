# 🎯 Action Prediction 분석

> **인용**: 논문 3페이지 2번째 줄부터 5페이지 1번째 줄까지의 Methodology 섹션

## 🎯 **Action Prediction 개요**

### **액션 예측의 목적**
> **인용**: 논문에서 제시된 액션 예측의 목적과 역할

- **로봇 제어**: 로봇의 동작을 예측하고 제어
- **의도 실행**: 사용자의 의도를 로봇 액션으로 변환
- **자율성**: 로봇의 자율적 의사결정

## 🏗️ **액션 예측 아키텍처**

### **입력 처리**
> **인용**: 논문에서 설명된 액션 예측의 입력 처리 방법

- **Vision Input**: 이미지 또는 비디오 입력
- **Language Input**: 자연어 명령 입력
- **History Input**: 과거 관측 및 액션 히스토리

### **특징 융합**
> **인용**: 논문에서 제시된 특징 융합 방법

- **Multi-modal Fusion**: Vision과 Language 특징 융합
- **Temporal Fusion**: 시간적 정보 융합
- **Context Integration**: 맥락 정보 통합

### **액션 출력**
> **인용**: 논문에서 설명된 액션 출력 구조

- **Action Space**: 연속 또는 이산 액션 공간
- **Action Dimension**: 액션의 차원 수
- **Action Range**: 액션 값의 범위

## 🔍 **액션 예측 방법론**

> **인용**: 논문에서 제시된 다양한 액션 예측 방법론

### **Policy Head 방식**
- **특징**: 각 히스토리 단계를 별도로 처리
- **장점**: 정확한 정보 융합
- **구조**: 별도의 정책 헤드에서 액션 예측

### **Interleaved 방식**
- **특징**: 시퀀스 전체를 하나의 모델로 처리
- **장점**: 전체적인 맥락 고려
- **구조**: 교차 모델링을 통한 액션 예측

### **One-step 방식**
- **특징**: 현재 상태만을 사용하여 액션 생성
- **장점**: 단순한 구조, 빠른 처리
- **구조**: 현재 관측 기반 액션 예측

## 📊 **액션 공간 설계**

> **인용**: 논문에서 제시된 액션 공간 설계 방법

### **Continuous Action Space**
- **특징**: 연속적인 값으로 액션 표현
- **장점**: 정밀한 제어 가능
- **단점**: 학습 복잡도 증가

### **Discrete Action Space**
- **특징**: 이산적인 값으로 액션 표현
- **장점**: 간단한 제어, 빠른 학습
- **단점**: 정밀도 제한
