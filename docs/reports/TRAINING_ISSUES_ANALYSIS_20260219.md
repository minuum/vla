# 학습 이슈 종합 분석 리포트

**작성일**: 2026-02-19  
**데이터 기반**: `basket_dataset_v2` (528 에피소드, 샘플 30개 분석)  
**학습 모델**: V2 Classification 9-Class (Epoch 2 Best)

## 1. 데이터셋 실제 행동 분석 (basket_dataset_v2 기반)

### 1-1. 9-Class 분포 (실측)

| 클래스 | 액션 | 프레임 수 | 비율 |
|:---|:---|:---:|:---:|
| **F** | 전진 (1.15, 0.0) | 261 | **49.9%** |
| **FL** | 전진+좌 (1.15, +1.15) | 87 | **16.6%** |
| **FR** | 전진+우 (1.15, -1.15) | 87 | **16.6%** |
| **L** | 좌회전 (0.0, +1.15) | 29 | 5.5% |
| **R** | 우회전 (0.0, -1.15) | 29 | 5.5% |
| **STOP** | 정지 (0.0, 0.0) | 30 | 5.7% |
| **B** | 후진 | 0 | 0.0% |
| **BL** | 후진+좌 | 0 | 0.0% |
| **BR** | 후진+우 | 0 | 0.0% |

### 1-2. 핵심 발견 (환각 없음)

**✅ 확인된 사실**:
- 로봇 경로는 **칼각(직각) 전진이 아닌 곡선형 S자 궤적**을 그립니다.
- 실제 에피소드 패턴: `F → R/L → FR/FL (대각선) → F → FL/FR → F → STOP`
- **대각선 액션(FL, FR) 비율이 33.3%**로 매우 높습니다.
- 고유 액션 값은 `{0.0, ±1.15}` 조합으로 이미 **이산화(Discrete)** 되어 있습니다.
- `B`, `BL`, `BR` 클래스는 실제 데이터에 **전혀 존재하지 않습니다**.

## 2. 식별된 이슈 목록

### 🔴 [해결 못함] Issue #1: 카메라 장면 변화에 모델이 지나치게 민감함

**증상**: 학습 데이터와 조금이라도 다른 장면(조명, 각도)에서 잘못된 액션 예측.  
**근본 원인**: 데이터셋이 특정 환경(학습 장면)에 치우쳐 있어 **시각적 일반화(Visual Generalization)** 능력 부족.  
**관련 수치**: 학습 완료 후 `train_acc=1.0`, `train_loss=0.0002` → 학습 데이터 **완전 암기** 신호.  
**완화 방법**: Epoch 3 → Epoch 2 체크포인트로 변경하여 과적합 수준을 낮춤 (완전 해결 아님).

---

### 🔴 [해결 못함] Issue #2: 9-Class 모델이 실제로는 5-Class만 학습됨

**증상**: `B`, `BL`, `BR` 3개 클래스가 데이터셋에 존재하지 않아 모델이 이를 전혀 학습하지 못함.  
**근본 원인**: 9-class 설계는 이론적이나, 실제 basket_dataset_v2는 5~6개 클래스만 사용.  
**위험성**: 실제 주행 시 후진이 필요한 상황에서 모델이 잘못된 클래스를 출력할 가능성 있음.

---

### 🔴 [해결 못함] Issue #3: 대각선 클래스(FL, FR)의 이해 부족 가능성

**증상**: 학습 데이터의 33.3%가 대각선 액션이나, 이 클래스들은 **동시에 두 방향의 시각적 신호**를 필요로 함.  
**우려**: 단순 분류 모델이 "목표물이 정면에 있을 때 전진"과 "목표물이 대각선에 있을 때 FL"을 구별하는 충분한 시각적 근거를 학습했는지 미검증.

---

### 🟡 [부분 해결] Issue #4: 과적합 (Overfitting)

**증상**: Epoch 3부터 `val_loss`가 0.024로 상승 (Epoch 2의 0.010 대비 2.4배).  
**Epoch 5**: `val_loss=0.454`로 완전한 발산.  
**조치**: Epoch 2 체크포인트 (`epoch=02, val_loss=0.010`)로 서버 전환 완료. ✅  
**미해결**: Early Stopping 패턴스가 3으로 설정되어 있어 다음 학습 시 자동 적용됨.

---

### 🟡 [부분 해결] Issue #5: 추론 서버 통합 로직 결함

**증상**: 모드 전환 코드 수정 중 `is_classification` 변수 미정의 오류 발생. 30분간 잘못된 결과 측정.  
**조치**: `self.mode == 'classification'`으로 수정 및 재배포 완료. ✅  
**영향**: 버그 기간 동안 측정된 성능 지표(Overall 5.6%) 무효화됨.

---

### 🟢 [해결] Issue #6: Regression 모델의 칼각 전진 문제

**증상**: 회귀 모델이 실제 물리적 경로(곡선)를 학습하지 못하고, loss 최소화를 위해 평균값(직진)만 출력.  
**조치**: 9-Class Classification 모델로 전환. 이산 클래스가 곡선 경로를 명확히 표현 가능. ✅  
**결과**: 단계별 스테이지 테스트에서 100% 정합성 확인.

---

### 🟢 [해결] Issue #7: 학습/추론 간 Instruction 언어 불일치

**증상**: 한국어 instruction으로 학습, 영어로 추론하여 모델 이해도 저하.  
**조치**: 영어 영어 instruction 통일 (`<grounding>An image of a robot ...`). ✅
