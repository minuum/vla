## 🔴 문제 요약

**증상**: 모델이 Left/Right 방향을 구분하지 못함
- LEFT 이미지 + LEFT 언어 → linear_y = +0.16
- LEFT 이미지 + RIGHT 언어 → linear_y = +0.16 (동일!)
- 언어 명령이 바뀌어도 예측이 바뀌지 않음

## 🔍 원인 분석

### 1. action_token이 거의 0

```python
action_token norm: 0.1412 (매우 작음)
action_token mean: 0.000006 (거의 0)
```

**의미**: `action_token`은 VLM output에서 action-relevant 정보를 추출하기 위한 학습된 query 토큰.
이 값이 0에 가까우면, VLM의 self-attention에서 다른 토큰(언어, 이미지)의 정보를 제대로 받지 못함.

### 2. 언어 정보가 Action Token에 도달하지 않음

VLM forward 경로:
```
[이미지 토큰] + [언어 토큰] + [action_token] → VLM → output_hidden_states
```

`action_token`이 마지막에 추가되고, VLM의 self-attention을 통해 앞의 토큰들(이미지+언어)의 정보를 받아야 함.
그러나 `action_token`이 거의 0이므로, query로서의 역할을 하지 못함.

### 3. Frozen Backbone의 한계

```python
freeze_backbone: True
lora_enable: True
```

- backbone이 frozen되어 있어서 self-attention 가중치가 고정됨
- LoRA가 적용되었지만, `action_token`이 학습되지 않은 상태에서는 효과가 제한적

## 💡 해결 방안

### 방안 A: action_token 초기화 개선

현재 `action_token`이 0으로 초기화되어 학습 초반에 gradient가 흐르지 않았을 가능성.

**수정**:
```python
# base_backbone.py에서
# 기존: self.action_token = nn.Parameter(torch.zeros(self.hidden_size))
# 수정: self.action_token = nn.Parameter(torch.randn(self.hidden_size) * 0.02)
```

### 방안 B: 이미지 특징에 언어 정보 직접 주입 (Language-Conditioned Feature)

Action Head에 언어 임베딩을 직접 concat하는 방식.

**수정 위치**: `forward_continuous`의 `action_hs` 생성 부분

```python
# 언어 임베딩의 평균 또는 [CLS] 토큰
lang_feature = output_hs[:, 0, :]  # 또는 attention pooling
# action_hs에 concat
action_hs = torch.cat([action_hs, lang_feature.unsqueeze(1).expand_as(action_hs)], dim=-1)
```

### 방안 C: Cross-Attention 추가 (Language → Action)

Action Head 앞에 cross-attention 레이어를 추가하여 언어 정보를 action feature에 명시적으로 주입.

### 방안 D: 이미지 토큰 전체 사용 (latent 증가)

현재 `latent=1`로 이미지 토큰 64개 중 1개만 사용.
더 많은 토큰을 사용하면 언어 정보가 포함된 토큰을 활용할 수 있음.

## 🚀 권장 우선순위

1. **방안 A** (빠른 수정): action_token 초기화 변경 후 재학습
2. **방안 B** (구조 변경): 언어 특징을 action_hs에 직접 concat
3. **방안 D** (설정 변경): latent 수 증가
