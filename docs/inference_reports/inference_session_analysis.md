# Inference Data & Model Result Analysis

## Background
최근 수집된 `docs/inference_reports/` 하위의 인퍼런스 에피소드 및 세션 데이터를 원격 저장소(`origin/vla-driving`)에 강제(force add) 병합 및 푸시 완료하였습니다. 동시에, 새롭게 수집된 데이터(총 15개 세션)가 어떤 형태로 구성되어 있고, 동일한 명령어에 대해 모델의 행동(Action)이 어떻게 차이를 보이는지 통계적 관점에서 비교 분석하였습니다.

## Analysis

### 1. Episode 및 Session 데이터 포맷
수집된 각각의 에피소드는 하나의 **JSON 메타데이터 파일**과 **이미지 디렉토리**의 하이브리드 형태로 구성되어 있습니다. `session_{timestamp}.json`의 구조는 아래와 같습니다:

- **Metadata Information**:
  - `session_id`: 고유 세션 ID (예: "20260305_100638")
  - `timestamp`: 세션 시작 시각
  - `model_name`: 사용 모델 (예: `mobile_vla_v3_exp07_lora`)
  - `instruction`: 사용자 입력 지시어 (예: "Navigate to the brown pot on the left")
  - `status`, `end_time`: 세션 종료 상태 및 시점
- **History (궤적 정보)**: 각 스텝(프레임) 단위의 세부 로그가 배열 형태로 기록됨
  - `step`: 프레임 (스텝) 번호
  - `timestamp`: 인퍼런스 및 행동 결정 시각
  - `action`: 모델이 예측한 최종 제어값 `[Linear Velocity, Angular Velocity]`
  - `latency_ms`: 인퍼런스 지연 시간 (단위: ms)
  - `chunk_preview`: 모델이 예측한 미래 궤적의 원시 복수 벡터 (Chunking Prediction)
  - `image_file`: 해당 프레임에서 수집된 카메라 이미지 경로 (예: `session_20260305_100638_images/frame_001.jpg`)
- **Summary**: `total_steps`, `avg_latency`, `last_action` 등 에피소드 요약

## 3. 20260306 세션 추가 분석: 목표 대상(브라운 팟) 회피 및 가시성 한계

가장 최근 수행한 `v3_exp08_center_goal` 테스트 세션(예: `20260306_113746` ~ `20260306_115040`)의 34스텝 연속 추론 로그를 분석한 결과, **특정 중간 프레임 구간에서 회피 조향을 멈추고 직진만 수행하는 치명적 한계**가 뚜렷하게 확인되었습니다.

- **조향 중단 현상 (Angular = 0.0)**: 에피소드 중반(약 10~16 스텝 구간)에 다다르면 연속적으로 예측되던 조향 값(-1.15 또는 1.15)이 갑자기 `0.0`으로 떨어지는 현상이 빈번히 발생합니다.
- **원인 (시야 이탈 - FOV Out)**: 로봇이 브라운 팟(장애물)에 가까워질수록 카메라의 낮은 상하 시야각(Vertical FOV) 한계로 인해 브라운 팟이 이미지 하단으로 빠르게 사라집니다. 대상이 시야에서 보이지 않게 되는 순간, 현재 이미지에만 의존하는 모델은 회피(조향) 행동을 멈추고 단순히 계속 직진하는 잘못된 판단을 내리게 됩니다.

## Findings

1. **안정적인 직진 제어**: Linear Action은 평균적으로 최대 가속치(1.15 근방)를 일정하게 유지합니다.
2. **시점(Mid-Frame) 의존적 조향 한계 (Reactive Behavior)**: 대상 객체 파악이 명확한 극초반부에는 좌/우 조향 방향을 잘 잡으나, 회피 기동 중간(Mid-Frame) 시점부터 물체가 시야(FOV) 밖으로 벗어나면 회전을 중단합니다. 이는 모델 구조상 메모리(Object Permanence)를 유지하지 못하기 때문입니다.
3. **고정 길이 프레임의 부작용**: 현재 추론/수집 단위가 특정한 고정 프레임 수(예: 18 또는 34 프레임)에 맞춰져 있어, 시야 확보와 무관하게 로직이 경직되어 있습니다.

## Proposed Data Collection Strategy & Next Steps

사용자 피드백과 로그 분석을 종합할 때, 장애물 회피를 온전히 마무리하기 위한 데이터셋 및 모델 구조 개선 전략을 다음과 같이 제안합니다:

| 개선 항목                             | 기존 방식                                  | 변경 제안 방향 (Next Step)                                                                                     | 기대 효과                                                                                         |
| :------------------------------------ | :----------------------------------------- | :------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------ |
| **Data Collection (데이터셋 다양화)** | 장애물이 정면에 보이는 단순 접근 경로 위주 | **중간/후반 프레임 집중 수집**: 근접해서 물체가 시야의 최하단에 있거나 살짝 벗겨지는 상황의 에피소드 대량 추가 | 객체가 안 보이더라도 이전에 보았던 맥락을 바탕으로 끝까지 궤적을 그리는 능력 확보                 |

## Conclusion

현재 로드된 `v3_exp08_center_goal` 모델은 목표물을 식별하고 초기 방향을 설정하는 데는 준수하나, **객체가 시야에서 이탈하는 중간(Mid) 프레임 대처 능력이 현저히 부족**합니다. 
따라서 향후 훈련에서는 1) 부분적 가려짐(Occlusion)이나 극단적 근접 시점의 시야 이탈 데이터를 집중 수집하고, 2) 단순 프레임 단위의 Reactive Action을 넘어, 에피소드의 길이를 유연하게 가져가며 맥락을 기억하는 연속형 예측(Continuous/Variable Length) 방식의 데이터 구성과 학습을 진행하는 것이 바람직합니다.
