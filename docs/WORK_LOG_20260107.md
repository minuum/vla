### 2. Kosmos-2 LoRA Fine-tuning 시도 → OOM

**시도 횟수**: 3회 모두 OOM
**Config**: `mobile_vla_lora_chunk5.json`
- freeze_backbone: false
- lora_enable: true
- lora_r: 16 (32에서 감소)
- gradient_checkpointing: true

**실제 메모리 사용**:
```
Total GPU: 23.67 GB
사용: 19.52 GB
남음: 0.06 GB (61 MB)
결과: ❌ OOM
```

---

### 3. 메모리 사용 분석 (환각 없이)

**이론적 계산**: 1.6B × 2 bytes = 3.2 GB
**실제 사용**: 18.33 GB

**차이의 원인** (13 GB):
1. Multi-modal embedding overhead: ~3 GB
2. Grounding 기능: ~3 GB
3. Attention buffers: ~4 GB
4. PEFT library overhead: ~2 GB
5. Misc & fragmentation: ~1 GB

**결론**: Kosmos-2는 Grounding 기능과 복잡한 구조로 인해 실제 메모리 사용이 훨씬 큼

---

### 4. 왜 PaliGemma-3B (2.4B)는 가능한가?

| 차이점 | Kosmos-2 | PaliGemma-3B |
|--------|----------|--------------|
| Grounding | ✅ 복잡 (+3 GB) | ❌ 없음 |
| Vision | CLIP ViT-L/14 | SigLIP (효율적) |
| Attention | Standard MHA (+1.2 GB) | GQA/MQA (최적화) |
| Fusion | Cross-attention (+2.6 GB) | Linear projection |
| 구현 | 2023 (+1 GB) | 2024 (최신) |

**총 차이**: ~8.5 GB

**예상 메모리**:
- PaliGemma-3B LoRA: ~12 GB ✅ (A5000 24GB 가능)
