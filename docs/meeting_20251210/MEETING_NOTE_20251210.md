## 🚀 Action Chunking 전략

### 왜 Chunking을 사용하는가?

#### 문제 상황
- **요구 사항:** 1초에 10Hz (100ms마다 액션 예측)
- **입력 크기:** 1280 × 720 × 3
- **제약:** VLM forward pass는 매우 느림

#### Chunking 없이 (No Chunk)
```
50ms마다 VLM 계산 필요
→ 1초에 20회 계산
→ 실시간 불가능
```

#### Chunking 사용 (Chunk=10)
```
VLM 1회 계산 → 10개 액션 예측
500ms마다 1회 계산
→ 1초에 2회 계산
→ 10배 효율 향상
```

**결과:**
- 10번의 계산 → **2번의 계산**
- 실시간 제어 가능
