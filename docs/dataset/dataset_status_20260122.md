## 3. 이슈 및 해결 사항
### 3.1. Basket 시나리오 통계 오류 수정
*   **증상**: Basket 시나리오(`basket_1box_left`) 수집 데이터가 Cup 시나리오(`1box_left`) 통계로 잘못 합산되는 현상 발생.
*   **원인**: 에피소드 이름 파싱 로직에서 `basket_` 접두사를 고려하지 않고 매칭하여, `1box_left`로 잘못 식별됨.
*   **조치**: `extract_scenario_from_episode_name` 함수 수정하여 `basket_` 시나리오를 우선 검색하도록 변경. 기존에 잘못 분류된 통계도 재정렬함.
*   **결과**: 현재 `basket_1box_left` 데이터가 올바르게 별도 집계되고 있음.

### 3.2. 자동 측정 로봇 멈춤 현상 수정
*   **증상**: Basket 시나리오 자동 측정 시작 직후 로봇이 움직이지 않는 현상.
*   **원인**: 에피소드 시작 시 시스템(카메라 등) 안정화 시간 부족으로 인한 `movement_lock` 경합.
*   **조치**: `execute_auto_measurement` 시작 부분에 2.0초 안정화 대기 시간 추가.
*   **결과**: 자동 측정이 정상적으로 시작되고 진행됨.

## 4. 향후 계획
1.  **Basket 시나리오 완료**: `basket_1box_left` 완료 후 나머지 Basket 시나리오(오른쪽, 2박스 등) 순차 진행.
2.  **데이터 검증**: 주기적으로 H5 파일 무결성 검증 수행 (현재 일부 파일 Truncated 경고 존재, 모니터링 필요).
