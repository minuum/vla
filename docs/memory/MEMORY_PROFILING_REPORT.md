# Phase 1 메모리 측정 결과 보고서

**날짜**: 2026-01-01  
**목적**: Jetson 로컬 온디바이스 VLA 실행을 위한 RAM 리소스 정확한 측정

---

## 📊 측정 결과 요약

### 구성 요소별 메모리 사용량 (FP16)

| 구성 요소 | 파라미터 | RAM 증가 | GPU | 로딩 시간 |
|-----------|----------|----------|-----|-----------|
| **Vision Encoder** | 0.30B | **+1.89GB** | 0.57GB | 4.7초 |
| **LLM (Text Model)** | 1.34B | **+3.59GB** | 2.51GB | 4.0초 |
| **전체 모델** | 1.66B | **+2.59GB** | 3.11GB | 4.8초 |
| **추론 Peak** | - | **+2.79GB** | 3.12GB | - |

### 핵심 발견

1. **통합 메모리 특성 확인**
   - Vision + LLM 개별: 1.89 + 3.59 = 5.48GB
   - 전체 모델: 2.59GB
   - **메모리 공유로 약 47% 절감!**

2. **Activation 오버헤드**
   - 로딩: 2.59GB
   - 추론 Peak: 2.79GB
   - **Activation: +0.20GB (8%)**

3. **여유 메모리**
   - 베이스라인: 4.7GB (OS + 기본 프로세스)
   - 추론 Peak: 7.49GB
   - **여유: 7.8GB (51%)**

## 💡 메모리 최적화 옵션

### Option 1: FP16 그대로 (권장)
```
현재 상태 유지
총 사용: ~9GB
여유: ~6GB
```

**장점**:
- 추가 작업 불필요
- 정확도 100% 유지
- 빠른 개발 가능

**단점**:
- 메모리 여유가 크지 않음

---

### Option 2: INT8 Vision Encoder (선택적)
```
Vision: 1.89GB → 0.95GB (-50%)
총 절감: ~1GB
여유: ~7GB
```

**장점**:
- 메모리 1GB 절감
- 정확도 98-99% 유지
- 구현 난이도 낮음

**단점**:
- 추가 개발 필요 (2-3일)
- quantization 검증 필요

---

### Option 3: INT8 Vision + INT4 LLM (비권장)
```
Vision: 1.89GB → 0.95GB
LLM: 3.59GB → 0.90GB
총 절감: ~3.5GB
여유: ~9.5GB
```

**장점**:
- 메모리 최대 절감
- 여유 9.5GB 확보

**단점**:
- 정확도 손실 (95-97%)
- 개발 시간 많이 소요 (1주+)
- BitsAndBytes 호환 문제
