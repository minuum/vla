## 발표 구조 (총 15분)

### 1부: 문제 정의 및 실험 설계 (3분)

**슬라이드 1: 이전 모델의 한계**
```
Case 4 (right_only) 문제점:
- Val Loss: 0.016 (높음)
- 데이터: 250 episodes (Right만)
- 방향 구분: 실패

연구 질문:
→ 데이터 2배 + Action Chunking 제거 = 성능 향상?
```

**슬라이드 2: 실험 설계**
```
변경사항:
1. 데이터: 250 → 500 episodes (Left+Right)
2. Action Chunk: fwd_pred_next_n 10 → 1
3. 기타: 동일 (Kosmos-2, LoRA, Window=8)

가설:
H1: 데이터 증가 → 일반화 향상
H2: Chunking 제거 → 학습 용이
H3: 결합 효과 → Loss 대폭 감소
```

**말할 내용**:
"기존 Case 4 모델은 Val Loss가 0.016으로 높고, Right 방향만 학습하여 일반화가 부족했습니다. 이를 해결하기 위해 두 가지 전략을 시도했습니다. 첫째, 데이터를 500개로 확대해 Left와 Right를 모두 포함시켰고, 둘째, Action Chunking을 10에서 1로 줄여 학습 난이도를 낮췄습니다."

  - Left+Right 분포 균형

  - Loss 30배 감소
  - 4 epochs 만에 수렴

  - 시너지 효과 확인

**말할 내용**:
"학습 결과, Epoch 4에서 Val Loss 0.000532로 최저점에 도달했고, Epoch 5에서 반등하여 과적합이 시작되었습니다. Early Stopping을 적용하여 Epoch 4를 최적 모델로 선정했습니다. Case 4와 비교하면 Val Loss가 30배 낮아졌고, 학습 시간도 60% 단축되었습니다. 세 가지 가설이 모두 검증되었습니다."

### 3부: Action Chunking 전략 분석 (4분)

**슬라이드 6: RoboVLMs 표준과의 차이**
```
RoboVLMs 원본:
- 모든 예제가 fwd_pred_next_n=10
- Calvin 환경 (7-DOF 조작)
- 복잡한 태스크

우리 환경:
- fwd_pred_next_n=1 (비표준)
- Mobile VLA (2D 네비게이션)
- 상대적으로 단순

전략적 선택: 환경에 맞춘 최적화
```

**슬라이드 7: Trade-off 분석**
```
Chunk=10 (표준):
  장점: 궤적 일관성, 계산 효율
  단점: 학습 어려움, 수렴 느림

Chunk=1 (우리):
  장점: 학습 쉬움 (30배), 실시간 반응
  단점: 떨림 위험, 일관성 부족

핵심 질문: 떨림이 실제로 발생하는가?
```

**슬라이드 8: 추론 테스트 결과**
```
Dummy 이미지 테스트:
- Left 명령: linear_y=+0.996 (정확)
- Right 명령: linear_y=-0.996 (정확)
- 추론 속도: 55-190ms (양호)
- GPU 메모리: 3.21GB (효율적)

결론: 방향 구분 정상 작동
```

**말할 내용**:
"RoboVLMs 표준은 Action Chunk를 10으로 사용하지만, 우리는 1을 선택했습니다. 이는 Mobile VLA의 단순한 2D 액션 공간에 더 적합하다고 판단했기 때문입니다. Trade-off는 학습 효율성과 떨림 위험입니다. Dummy 이미지 테스트에서는 방향 구분이 정확했고, 추론 속도도 양호했습니다. 하지만 실제 떨림 발생 여부는 로봇 테스트가 필요합니다."

### 4부: 배포 전략 및 리스크 관리 (3분)

**슬라이드 9: 단계적 배포 계획**
```
Phase 1: 성능 검증 (완료)
  - Epoch 4 선정
  - Dummy 테스트 통과

Phase 2: 정확도 평가 (진행중)
  - 100 episodes 테스트
  - 목표: 95% 이상

Phase 3: 로봇 실증 (예정)
  - 떨림 측정
  - 경로 추종 평가

Phase 4: 최종 결정 (조건부)
  - 성공 → Epoch 4 확정
  - 떨림 → Smoothing 적용
  - 실패 → Case 4 롤백
```

**슬라이드 10: 리스크 대응**
```
위험 1: 떨림 (Jittering)
  대응: EMA Smoothing / 임계값 / Case 4 롤백

위험 2: 방향 정확도 부족
  대응: abs_action 전략 추가

위험 3: 예상치 못한 문제
  대응: Case 4 준비 완료 (즉시 전환 가능)

핵심: 다층 안전망 구축
```

**말할 내용**:
"배포는 단계적으로 진행합니다. 현재 Phase 1은 완료했고, Phase 2는 진행 중입니다. 주요 리스크는 떨림과 방향 정확도인데, 각각 Smoothing과 abs_action으로 대응 가능합니다. 최악의 경우 Case 4로 롤백할 수 있어 리스크는 제한적입니다."

## 질의응답 대비

**예상 질문 1**: "No Chunk가 비표준인데 괜찮은가?"

**답변**:
"RoboVLMs는 7-DOF 조작 태스크용이고, 우리는 2D 네비게이션입니다. 액션 공간 복잡도가 다르므로 환경에 맞게 최적화했습니다. Val Loss 30배 향상이 이를 뒷받침합니다. 다만 떨림 위험이 있어 로봇 테스트가 필수이며, 문제 발생 시 표준 방식으로 전환 가능합니다."

**예상 질문 2**: "떨림이 실제로 발생할 가능성은?"

**답변**:
"이론적으로는 가능하지만, 세 가지 이유로 제한적일 것으로 예상합니다. 첫째, Mobile VLA 로봇의 물리적 관성이 자연스러운 Smoothing 역할을 합니다. 둘째, 액션이 속도 명령이므로 즉각 반영되지 않습니다. 셋째, 필요 시 EMA를 적용할 수 있습니다. 실제 발생 여부는 로봇 테스트로 확인하겠습니다."

**예상 질문 3**: "Case 4와 실제 성능 비교는?"

**답변**:
"Val Loss는 30배 낮지만, 실제 태스크 성공률은 별도 측정이 필요합니다. 현재 Dummy 테스트에서는 방향 구분이 정확했지만, 복잡한 환경에서의 성능은 로봇 실증으로 확인하겠습니다. 정량적 비교를 위해 동일한 경로에서 두 모델을 테스트할 계획입니다."

**예상 질문 4**: "500 episodes로 충분한가?"

**답변**:
"Epoch 4에서 Val Loss가 최저점을 찍고 Epoch 5에서 과적합이 시작되었으므로, 현재 데이터로 충분히 수렴했습니다. 추가 데이터는 필요하지 않으며, 오히려 Early Stopping이 적절히 작동한 좋은 사례입니다. 향후 성능 향상이 필요하면 모델 구조 개선을 우선 고려하겠습니다."

**예상 질문 5**: "다음 연구 계획은?"

**답변**:
"단기적으로는 Hybrid Chunking을 연구할 계획입니다. 직선 구간에서는 Chunk=10으로 효율성을 높이고, 장애물이나 회전 구간에서는 Chunk=1로 반응성을 확보하는 방식입니다. 중기적으로는 모델 경량화와 데이터 확대, 장기적으로는 Multi-Task Learning을 고려하고 있습니다."
