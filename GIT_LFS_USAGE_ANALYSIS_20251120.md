## 문제 분석

### Git LFS Bandwidth 초과 원인
1. **H5 파일 다운로드**: 237개 H5 파일 (약 6.1GB)을 여러 번 fetch/checkout
2. **체크포인트 파일**: 체크포인트는 LFS에 없지만, 다른 파일들이 bandwidth 소비
3. **다중 브랜치**: 여러 브랜치에서 LFS 파일 fetch 시 중복 다운로드

### 해결 방안

#### 즉시 조치
1. **로컬 LFS 캐시 활용**: 이미 다운로드된 파일은 재사용
   ```bash
   git lfs fetch --all  # 이미 완료됨
   git lfs checkout     # 이미 완료됨
   ```

2. **필요한 파일만 선택적 다운로드**
   ```bash
   git lfs pull --include="*.h5"  # H5 파일만 선택적 다운로드
   ```

#### 장기 조치
1. **LFS 예산 증가**: GitHub Pro/Team 플랜으로 업그레이드
2. **데이터 분리**: 대용량 파일을 별도 저장소로 분리
3. **11일 후 리셋 대기**: 무료 할당량이 자동으로 리셋됨
