# 📊 ABC 분할 (ABC Split) 분석

> **인용**: 논문 Table II의 ABC 분할 훈련 결과

## 🎯 **ABC 분할 개요**

### **ABC 분할의 정의**
ABC 분할은 CALVIN 벤치마크에서 제한된 훈련 데이터 상황에서 모델의 일반화 능력을 평가하기 위해 설계된 데이터셋 구성 방식입니다.

### **분할 구성**
- **A**: 훈련 데이터에 포함된 작업과 동일한 작업을 포함하되, 새로운 환경이나 조건에서 수행되는 테스트 데이터
- **B**: 훈련 데이터에 포함되지 않은 새로운 작업을 포함하는 테스트 데이터
- **C**: 훈련 데이터에 포함된 작업과 동일한 작업이지만, 새로운 도구나 객체를 사용하는 테스트 데이터

### **ABCD 분할과의 차이점**
- **훈련 데이터 제한**: D 분할이 제외되어 훈련 데이터가 더 제한적
- **일반화 능력 평가**: 더 어려운 조건에서의 모델 성능 평가
- **데이터 효율성**: 제한된 데이터로 얼마나 잘 학습할 수 있는지 평가

## 🔍 **ABC 분할에서의 의미**

### **데이터 효율성 평가**
ABC 분할은 제한된 훈련 데이터 상황에서 모델이 얼마나 효율적으로 학습할 수 있는지를 평가합니다.

### **VLA의 데이터 효율성 우위**
- **제한된 데이터에서도 우수한 성능**: ABC 분할에서 KosMos P.H.의 압도적 성능
- **일반화 능력**: 새로운 작업과 환경에 대한 강건한 적응성
- **학습 효율성**: 적은 데이터로도 높은 성능 달성

### **RoboVLMs의 데이터 효율성**
- **압도적 성능 차이**: GR-1 대비 모든 지표에서 큰 폭 향상
- **연속 작업에서의 안정성**: 작업 수가 증가해도 상대적 우위 유지
- **강건한 일반화**: 제한된 훈련 데이터로도 우수한 일반화 능력

### **데이터 제한의 영향**
- **단순 작업**: 제한된 데이터로도 높은 성능 가능
- **복잡한 연속 작업**: 더 많은 훈련 데이터가 필요
- **일반화 능력**: 충분한 데이터가 복잡한 시나리오에 중요

## 🎯 **ABC 분할의 핵심 인사이트**

### **VLA 아키텍처의 중요성**
- **Policy Head + Continuous Action**: 제한된 데이터에서도 우수한 성능
- **KosMos 백본**: 데이터 효율성과 일반화 능력의 우수성
- **RoboVLMs 프레임워크**: 체계적 설계의 효과

### **실용적 시사점**
- **데이터 수집 비용**: 제한된 데이터로도 효과적인 VLA 구축 가능
- **빠른 프로토타이핑**: 적은 데이터로 초기 성능 검증 가능
- **확장성**: 점진적 데이터 증가에 따른 성능 향상 가능
